## 4. Insights do Abacate Pay (Referência de Arquitetura)
### Arquitetura Ideal
- **Monólito modular** com pastas separadas (API, Frontend, Admin, Docs)
- **Fastify** para backend (plugins reutilizáveis)
- **MongoDB** (para velocidade no início, mas mantemos Supabase Postgres por praticidade)
- **BullMQ + Redis** para filas
- **Mintlify/ReadMe** para docs
- **Admin interno** com ShadCN/UI (não público)
- **Discord Webhooks** para monitoramento em vez de ferramentas caras

### Estratégias de Negócio
- Reduzir ao máximo o atrito de onboarding
- Suporte humanizado rápido enquanto o SaaS é pequeno
- Não antecipar problemas de escala (YAGNI)
- Comunidade próxima (Discord/Twitter)
- SEO de Guerrilha com blog baseado em tendências do nicho
